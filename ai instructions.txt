1. --- Stack ---
* В проекте есть файл `docker-compose`
* Репозиторий на GitHub: [Aleksey095599/alligator-market](https://github.com/Aleksey095599/alligator-market)
* Backend на Spring Boot
* Frontend на Angular (TypeScript)
* Разработка ведётся в IntelliJ IDEA Ultimate

2. --- Общий принцип ответов ---
Всегда старайтесь искать максимально простые решения, но с учётом возможных точек роста. Минимизируйте количество кода и аннотаций, делая структуру ясной и понятной. Простота обеспечивает надёжность, а чёткая организация — гибкость для масштабирования.

3. --- Триггерная фраза «решить задачу» ---
Если в сообщении пользователя есть фраза «решить задачу» и приводится описание этой задачи то следовать инструкциям А и Б.
A) Инструкция для размышления модели ChatGPT:
* Максималистичное решение: Проанализируйте лучшие мировые практики в крупных успешных проектах, решающих аналогичные задачи. Выберите наиболее качественные подходы и архитектуры.
* Минимально-достаточное решение: Найдите самое простое и лаконичное решение (KISS), полностью удовлетворяющее текущим требованиям.
* Итоговое решение: Разработайте решение, сочетающее простоту и лаконичность минимально-достаточного подхода с возможностью расширения до максималистичного решения при необходимости.
Б) Формат ответа:
* Максималистичное решение: Кратко опишите без кода.
* Минимально-достаточное решение: Кратко опишите без кода.
* Итоговое решение: Кратко опишите без кода, а затем задайте подробные шаги для реализации решения.

4. --- Правила оформления ---
Общий принцип: Оформлять код согласно лучшим мировым практикам применяя минималистичный подход — кратко, без лишнего шума.
Инструкции оформления:
* Добавляйте комментарии (на русском языке, кроме устоявшихся терминов) НАД объектами, чтобы объяснить их назначение (например, `/* Entity для валютной пары. */`)
* ВНУТРИ объектов добавляйте краткие комментарии (на русском языке, кроме устоявшихся терминов) там где это важно для понимания логики кода (например, `// Сохраняем entity в БД`)
* Допускается для наглядности применять в комментариях стрелочки или иные символы если это уместно и считается хорошей практикой
* Текст логов, сообщения об ошибках, внешние тексты и тому подобное оформляйте на английском языке согласно лучшим стандартам

5. --- Инструкция для работы с холстом ---
* Пишите на холсте только по моей просьбе
* Вносите правки на холсте только по моей просьбе и только в указанных местах

6.  --- Триггерная фраза «обнови дерево проекта» ---
Если пользователь просит «обнови дерево проекта» (или формулирует аналогичную просьбу), ChatGPT должен:
* Выполнить запрос к GitHub API GET /repos/<owner>/<repo>/git/trees/main?recursive=1
* Отфильтровать полученный JSON, оставив только элементы, у которых path начинается с "src/main/"
* Использовать этот JSON-массив как актуальную структуру проекта при последующем анализе и ответах
* Выводить полученную структуру проекта в чат для пользователя не обязательно, при успешном получении структуры напишите: «дерево проекта обновлено»
