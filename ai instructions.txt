1. **Stack:**
* В проекте есть файл `docker-compose`;
* Репозиторий на GitHub: [Aleksey095599/alligator-market](https://github.com/Aleksey095599/alligator-market);
* Backend на Spring Boot;
* Frontend на Angular (TypeScript);
* Разработка ведётся в IntelliJ IDEA Ultimate.

2. **Основной принцип:**
Всегда старайтесь сделать решение максимально простым, но с учётом возможных точек роста. Минимизируйте количество кода и аннотаций, делая структуру ясной и понятной. Простота обеспечивает надёжность, а чёткая организация — гибкость для масштабирования.

3. **Инструкция для ChatGPT:**
Если пользователь начинает сообщение со слова «Задача», выполните следующие шаги:
* **Максималистичное решение:** Проанализируйте лучшие мировые практики в крупных успешных проектах, решающих аналогичные задачи. Выберите наиболее качественные подходы и архитектуры.
* **Минимально-достаточное решение:** Найдите самое простое и лаконичное решение (KISS), полностью удовлетворяющее текущим требованиям.
* **Итоговое решение:** Разработайте решение, сочетающее простоту и лаконичность минимально-достаточного подхода с возможностью расширения до максималистичного решения при необходимости.

**Формат итогового ответа:**
* **Максималистичное решение:** Кратко опишите подход без кода.
* **Минимально-достаточное решение:** Кратко опишите подход без кода.
* **Итоговое решение:** Кратко опишите подход общими словами (без кода), а затем задайте подробные шаги для реализации подхода.

4. **Правила оформления:**
Цель оформления — не документация, а пояснения для разработчиков. Избегайте лишних комментариев, они должны быть краткими и лаконичными.
* Добавляйте краткие комментарии (на русском языке, кроме устоявшихся терминов) над объектами, чтобы объяснить их назначение (например, `/* Entity для валютной пары. */`). Не комментируйте рутинные и шаблонные объекты (например, конструкторы внутри классов и т. д.).
* Внутри объектов добавляйте краткие комментарии (на русском языке, кроме устоявшихся терминов), где это необходимо для понимания логики кода (например, `// Сохраняем entity в БД`). Для наглядности можно добавлять стрелки и другие символы.
* Текст логов, сообщения об ошибках, внешние тексты и т. д. оформляйте на английском языке согласно лучшим практикам для формулировок.

5. **Инструкция для работы с холстом:**
* Пишите на холсте только по моей просьбе.
* Вносите правки на холсте только по моей просьбе и только в указанных местах.
